# 架构

## 边界

`src/tasks/ChaosTask.py` 是唯一了解 `ok-script` API 的模块。文件内的 `ChaosTask` 负责取得一帧 OCR、
把框架的 `Box` 转换为 `ScreenContext`，以及执行决策核心返回的单个动作；`CardCollectorTask` 是一次性
采集适配器，只保存一帧脱敏图像和这一帧的 OCR 证据。页面规则和卡牌核心不得直接调用鼠标、键盘、
睡眠或再次 OCR。

```text
ok-script
   │ 一帧 OCR
   ▼
ChaosTask（适配层）
   │ ScreenContext
   ▼
ChaosEngine
   ├─ 所有 PageHandler 同时评分
   ├─ 置信度和显式优先级选页
   ├─ 歧义保护
   ├─ 重复动作冷却
   └─ 最多产生一个 Action
   │
   ▼
TaskActionExecutor → ok-script 点击/按键
```

卡牌采集走一条无操作旁路：

```text
ok-script 一帧画面 → 隐私遮挡 → 同帧 OCR → CardSampleWriter → pending 样本
```

采集任务不调用动作执行器，也不把 OCR 文本自动写入正式卡牌目录。

实时卡牌识别同样走无操作旁路，但不写入文件：

```text
ok-script 内存画面 → 单次 OCR → ScreenContext → RuntimeCardIndex
                                                   ├─ FlashRecognition
                                                   ├─ CardObservation
                                                   └─ 审核目录准入 → MaterializedCard 或拒绝
```

“识别当前卡牌闪光（只读）”只读取 `self.frame` 和调用一次 `ocr(frame=...)`。完整画面仅在本轮内存中用于
确认文字相对位置和神闪金色徽记，不调用截图保存、鼠标或键盘 API。运行时索引在源码模式下从仓库读取
两个候选 JSON 和 `data/cards` 审核目录；单文件构建只打包这些结构化资料，不打包开发样本图片。

为降低实时延迟，OCR 只处理词库中配置的卡牌详情相对区域；OCR 返回的位置仍换算到完整画面坐标，金色徽记
判断继续使用完整内存帧。只读索引在任务对象第一次运行时加载一次并复用，信息面板会分别显示 OCR、匹配和
总耗时，并单独展示识别结果是否已通过审核目录准入。候选资料可以改善识别，但不能被策略层当作批准事实。

## 一轮运行的不变量

1. `ChaosTask.run()` 只调用一次 `ocr()`。
2. 页面处理器只能读取同一个不可变 `ScreenContext`。
3. 引擎评估全部处理器，处理器在注册表中的先后顺序不决定结果。
4. 页面歧义时不操作。
5. 一个 `Decision` 最多包含一个 `Action`。
6. 动作完成后结束本轮，下一轮使用新画面验证结果。
7. 开关任务会清空所有临时状态，禁止跨运行继承页面状态。

一次卡牌采集同样只允许一次画面读取和一次 OCR，但动作数固定为零。
一次实时卡牌识别也只允许一次画面读取和一次 OCR，动作数和文件写入数都固定为零。

## 新增页面

新增页面处理器需要实现：

- 全局唯一的 `page_id`；
- 显式 `priority`；
- 无副作用的 `match()`；
- 无副作用的 `plan()`；
- 至少一组命中测试和一组相似页面不误命中的测试。

文字规则优先使用页面标题与按钮的组合。仅凭“确认”“继续”等通用词不能形成高置信度页面。
需要颜色、模板或区域信息时，应先扩展 `ScreenContext` 的证据模型，不得在处理器中重新截图。

## 安全策略

- 初始配置保留存档。
- 通用确认处理器排除删除、覆盖、重置等关键词。
- 删除存档等不可逆操作必须具有专用模板、专用页面处理器和二次验证。
- 未知页面只能记录或保存脱敏截图，不能随机点击。
- 启动时过滤 `ok-script` 输出环境变量的日志记录，避免敏感配置进入日志。
